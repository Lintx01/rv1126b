# 手势模型接入说明

## 模型信息

当前接入的手势模型路径：

```text
model/yolov5_gesture_rv1126b.rknn
```

模型类型：

```text
YOLOv5 classification / cls
```

输入尺寸：

```text
224x224
```

输出类别：

```text
0~14 共 15 类
```

其中 0~9 是数字手势，另外 5 类是其他手势。当前系统只需要把其中部分类别映射成系统动作。

## 当前动作映射

当前代码在 `src/GestureModel.cpp` 中集中定义了类别 ID：

```cpp
constexpr std::size_t kGestureNoneClassId = 0;
constexpr std::size_t kGestureStartClassId = 1;
constexpr std::size_t kGestureStopClassId = 2;
constexpr std::size_t kGestureHeartClassId = 3;
```

当前暂定：

```text
class 0 -> None
class 1 -> Start
class 2 -> Stop
class 3 -> Heart
其他类别 -> None，但保留 gesture_name = class_x
```

后续如果训练标签顺序变化，只需要改上面的 ID 常量。

## Heart 的含义

`Heart` 在当前工程里表示“比心/爱心手势”事件，不是心率。识别到后会触发：

```text
GestureType::Heart
-> AppState gesture_name = "heart"
-> DisplayFace::GESTURE_OK_FACE
-> DisplayDevice::showHeartExpression()
```

## 输出解析逻辑

`GestureModel::parseOutput()` 当前按分类模型处理：

1. 读取第一个输出 tensor。
2. 最多取前 15 个类别分数。
3. 如果输出已经像 softmax 概率，就直接使用。
4. 如果输出像 logits，就先做 softmax。
5. 取最高分类别。
6. 分数低于 `gesture_score_threshold` 时不触发动作。
7. 分数达标后按映射表转成 `Start`、`Stop`、`Heart` 或 `None`。

## 主机 fallback 自测

没有 RKNN runtime 时，模型会进入 fallback。为了让电脑 mock 测试能稳定跑通流程，fallback 现在按推理次数触发：

```text
第 1 次手势推理 -> Start
第 5 次手势推理 -> Heart
第 12 次手势推理 -> Stop
```

这样可以避免按固定 `frame_id` 触发时，被 AI 调度跳帧错过。
