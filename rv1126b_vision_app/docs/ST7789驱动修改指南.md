# ST7789 驱动修改指南

## 1. 当前代码位置

ST7789 相关代码在：

```text
include/Interfaces.hpp
src/DisplayDevice.cpp
src/main.cpp
```

当前 `DisplayDevice.cpp` 已经提供 Linux `spidev` + `/sys/class/gpio` 的基础实现，并保留非 Linux 桩路径：

- `open()`：打开 SPI、配置 GPIO、初始化屏幕。
- `resetPanel()`：控制 RESET 引脚复位。
- `initPanel()`：发送 ST7789 初始化命令。
- `drawRgb565Bitmap()`：发送 RGB565 图片数据。
- `showHeartExpression()`：比心手势触发表情显示。

## 2. 需要按硬件修改的配置

在 `src/main.cpp` 修改：

```cpp
config.st7789_spi_device = "/dev/spidev0.0";
config.st7789_spi_speed_hz = 40000000;
config.st7789_width = 240;
config.st7789_height = 240;
config.st7789_rotation = 0;
config.st7789_invert_color = true;
config.st7789_gpio_dc = 24;
config.st7789_gpio_reset = 23;
config.st7789_gpio_backlight = 25;
config.display_timezone_offset_minutes = 480;
```

字段说明：

- `st7789_spi_device`：SPI 设备节点。
- `st7789_spi_speed_hz`：SPI 频率，常见 24MHz 到 40MHz。
- `st7789_width` / `st7789_height`：屏幕分辨率，常见 240x240、240x320。
- `st7789_rotation`：显示方向，通常取 0、90、180、270。
- `st7789_invert_color`：是否开启反色，很多 ST7789 模组需要开启。
- `st7789_gpio_dc`：数据/命令选择引脚。
- `st7789_gpio_reset`：硬复位引脚。
- `st7789_gpio_backlight`：背光控制引脚。
- `display_timezone_offset_minutes`：屏幕时钟使用的 UTC 偏移，默认 `480`，即北京时间 UTC+8。

GPIO 编号必须按 RV1126B 的实际 Linux GPIO 编号填写，不一定等于原理图上的管脚编号。

## 3. SPI 驱动修改步骤

如果需要把当前 `/sys/class/gpio` 实现改成 `libgpiod`，或适配厂商专用 SPI/GPIO 驱动，可从 `DisplayDevice::open()` 开始调整：

1. 打开 SPI 设备：

```cpp
int spi_fd = open(config.st7789_spi_device.c_str(), O_RDWR);
```

2. 配置 SPI：

```cpp
uint8_t mode = SPI_MODE_0;
uint8_t bits = 8;
uint32_t speed = config.st7789_spi_speed_hz;
ioctl(spi_fd, SPI_IOC_WR_MODE, &mode);
ioctl(spi_fd, SPI_IOC_WR_BITS_PER_WORD, &bits);
ioctl(spi_fd, SPI_IOC_WR_MAX_SPEED_HZ, &speed);
```

3. 写命令和数据：

```cpp
setDc(false);
write(spi_fd, &cmd, 1);

setDc(true);
write(spi_fd, data, len);
```

建议封装：

```cpp
bool writeCommand(uint8_t cmd);
bool writeData(const uint8_t* data, size_t len);
```

## 4. GPIO 驱动修改步骤

推荐使用 `libgpiod`，比 `/sys/class/gpio` 更适合新内核。

需要实现：

```cpp
bool setDc(bool data_mode);
bool setReset(bool high);
bool setBacklight(bool on);
```

复位时序：

```text
RESET=0，等待 10ms
RESET=1，等待 120ms
```

背光：

```text
初始化完成后 BL=1
关闭屏幕前 BL=0
```

## 5. ST7789 初始化命令

常见初始化顺序：

```text
0x11 SLPOUT  退出睡眠，延时 120ms
0x3A COLMOD  设置像素格式，RGB565 参数 0x55
0x36 MADCTL  设置方向和 RGB/BGR 顺序
0x21 INVON   开启反色，按屏幕模组决定是否需要
0x29 DISPON  开启显示，延时 20ms
```

`MADCTL` 参数需要按屏幕方向调：

```text
0 度：   0x00 或 0x08
90 度：  0x60 或 0x68
180 度： 0xC0 或 0xC8
270 度： 0xA0 或 0xA8
```

如果颜色红蓝反了，通常是 RGB/BGR 位不对，调整 `MADCTL` 的 `0x08` 位。

## 6. RGB565 表情图片替换

当前 `showHeartExpression()` 使用 2x2 示例像素。实际项目建议：

1. 准备一张 240x240 PNG。
2. 离线转换为 RGB565 C 数组。
3. 替换 `heart_pixels`。
4. 调用：

```cpp
drawRgb565Bitmap(heart_pixels, 240, 240);
```

离线转换的好处：

- 运行期不需要 PNG/JPEG 解码库。
- 表情显示延迟更低。
- 内存占用更可控。

## 7. LVGL 空闲时钟测试模式

当前 `AppConfig::lvgl_idle_only_test` 默认为 `true`。在这个模式下，LVGL 只显示空闲时钟页；业务侧投递的表情事件会被显示层忽略，不会切换到坐姿、喝水或手势表情。

日志会显示：

```text
[Display][LVGL] 空闲时钟测试模式
[Display][LVGL] 空闲时钟测试模式：忽略表情事件
```

这是预期行为，用于只验证 LVGL 时钟页和 ST7789 刷屏链路。如果后续需要启用真实表情切换，需要把 `lvgl_idle_only_test` 设为 `false`，并确认对应表情页或图片资源已经接好。

空闲时钟页不再依赖板端本地时区配置。显示层会用系统 epoch 时间加 `display_timezone_offset_minutes` 后再拆分年月日时分秒；默认值 `480` 会固定显示北京时间。日志会显示：

```text
[Display][Clock] 使用北京时间 UTC+8，屏幕时区偏移=480 分钟
```

## 8. 常见问题

### 屏幕全黑

- 检查背光 GPIO 是否拉高。
- 检查 RESET 时序。
- 检查 SPI 设备节点是否正确。
- 检查 DC 引脚是否接反。

### 颜色异常

- 检查 RGB565 字节序。
- 调整 `MADCTL` 的 RGB/BGR 位。
- 尝试切换 `st7789_invert_color`。

### 图像偏移

- 检查屏幕模组是否有 X/Y offset。
- 某些 240x240 ST7789 需要在 CASET/RASET 中加偏移。

### 刷屏慢

- 提高 SPI 频率，例如 24MHz 到 40MHz。
- 分块写入，减少频繁切换 DC。
- 表情图使用 RGB565，避免 RGB888。

