# ST7789 实机点屏前检查清单

本文档用于 RV1126B 开发板连接 ST7789 屏幕前的板端检查。当前计划接线如下：

```text
ST7789 VCC -> P16 3.3V
ST7789 GND -> P16 GND
ST7789 SCL -> P16 Pin23 SPI1_CLK
ST7789 SDA -> P16 Pin19 SPI1_MOSI
ST7789 CS  -> P16 Pin24 SPI1_CSN0
ST7789 DC  -> P16 Pin13 GPIO4_A0
ST7789 RES -> P16 Pin15 GPIO0_C7
ST7789 BLK -> 3.3V 常亮
```

## 1. 检查 `/dev/spidev`

在 RV1126B 板子上执行：

```bash
ls -l /dev/spidev*
ls -l /sys/class/spidev/
ls -l /sys/bus/spi/devices/
dmesg | grep -i spi
dmesg | grep -i spidev
```

如果 SPI1 CS0 已正确启用，通常应看到类似：

```text
/dev/spidev1.0
/sys/bus/spi/devices/spi1.0
```

本接线使用 `SPI1_CLK / SPI1_MOSI / SPI1_CSN0`，因此优先预期设备节点是：

```text
/dev/spidev1.0
```

## 2. 确认是 `/dev/spidev1.0` 还是 `/dev/spidev0.0`

查看设备树 alias：

```bash
ls /proc/device-tree/aliases
for f in /proc/device-tree/aliases/spi*; do echo "$f"; strings "$f"; done
```

查看 SPI 设备：

```bash
ls -l /sys/bus/spi/devices/
readlink -f /sys/class/spidev/spidev*
```

判断规则：

```text
SPI1_CSN0 -> spi1.0 -> /dev/spidev1.0
SPI0_CSN0 -> spi0.0 -> /dev/spidev0.0
```

当前接线使用 SPI1，因此代码中的 `st7789_spi_device` 后续大概率应配置为：

```cpp
config.st7789_spi_device = "/dev/spidev1.0";
```

## 3. 确认 DC、RES 的 Linux GPIO 编号

Rockchip 常见 GPIO 编号计算方式：

```text
GPIOx_Ay = x * 32 + y
GPIOx_By = x * 32 + 8 + y
GPIOx_Cy = x * 32 + 16 + y
GPIOx_Dy = x * 32 + 24 + y
```

当前接线：

```text
DC  -> GPIO4_A0 -> 4 * 32 + 0 = 128
RES -> GPIO0_C7 -> 0 * 32 + 16 + 7 = 23
```

初步配置应为：

```text
gpio_dc    = 128
gpio_reset = 23
```

板端确认命令：

```bash
ls /sys/class/gpio
cat /sys/kernel/debug/gpio
```

如果 `/sys/kernel/debug/gpio` 不存在，先挂载 debugfs：

```bash
mount -t debugfs none /sys/kernel/debug
cat /sys/kernel/debug/gpio
```

也可以手动导出测试：

```bash
echo 128 > /sys/class/gpio/export
echo out > /sys/class/gpio/gpio128/direction
echo 1 > /sys/class/gpio/gpio128/value
echo 0 > /sys/class/gpio/gpio128/value

echo 23 > /sys/class/gpio/export
echo out > /sys/class/gpio/gpio23/direction
echo 1 > /sys/class/gpio/gpio23/value
echo 0 > /sys/class/gpio/gpio23/value
```

注意：如果某个 GPIO 已经被内核驱动占用，导出或设置方向可能失败。不要在不确定复用关系时反复拉高/拉低。

## 4. 如果没有 spidev，设备树修改方向

如果执行 `ls /dev/spidev*` 没有任何结果，需要检查设备树是否启用了 SPI1 和 spidev 子节点。

设备树方向如下，具体 pinctrl 名称以当前 SDK 为准：

```dts
&spi1 {
    status = "okay";
    pinctrl-names = "default";
    pinctrl-0 = <&spi1m0_clk &spi1m0_mosi &spi1m0_cs0>;

    spidev@0 {
        compatible = "rockchip,spidev";
        reg = <0>;
        spi-max-frequency = <24000000>;
    };
};
```

关键点：

```text
启用 spi1
启用 cs0
打开 clk/mosi/csn0 pinmux
挂 spidev@0
status = "okay"
```

修改后需要重新编译 dtb，替换板端实际启动使用的 dtb，然后重启确认 `/dev/spidev1.0` 是否出现。

## 5. 第一次测试 SPI 速度建议

第一次点屏不要直接使用 `40MHz`。建议从低速开始：

```text
4MHz 或 8MHz
```

稳定后再逐步提高：

```text
8MHz -> 16MHz -> 24MHz -> 32MHz -> 40MHz
```

第一次建议代码配置为：

```cpp
config.st7789_spi_speed_hz = 8000000;
```

原因：ST7789 模块、杜邦线长度、板端驱动能力、地线质量都会影响高频 SPI 稳定性。

## 6. 点屏失败排查顺序

1. 电源  
   确认 VCC 是 `3.3V`，GND 共地。不要接 5V，除非模块明确支持。

2. 背光  
   当前 `BLK -> 3.3V`，背光应常亮。如果屏完全黑，先查 BLK、VCC、GND。

3. SPI 设备  
   确认存在 `/dev/spidev1.0`。如果代码还在打开 `/dev/spidev0.0`，会直接失败。

4. SPI 接线  
   模块标注一般为：

   ```text
   SCL -> SPI CLK
   SDA -> SPI MOSI
   CS  -> SPI CS
   ```

   ST7789 通常不需要 MISO。

5. DC/RES GPIO  
   确认 `GPIO4_A0 = 128`、`GPIO0_C7 = 23` 能导出、能设置方向、能拉高拉低。

6. RES 复位时序  
   RES 必须能被代码拉低再拉高。RES 接错时，即使 SPI 有波形也可能不显示。

7. SPI mode  
   ST7789 常用 `SPI_MODE_0`。当前代码方向也是 mode 0。

8. SPI 速度  
   先降到 `4MHz` 或 `8MHz` 测试。

9. 显示参数  
   如果能亮但花屏，重点查 RGB565 字节序、MADCTL、屏幕偏移、分辨率、颜色反转。

## 7. 当前代码配置建议

按当前接线，后续谨慎修改方向如下：

```cpp
config.st7789_spi_device = "/dev/spidev1.0";
config.st7789_spi_speed_hz = 8000000;
config.st7789_gpio_dc = 128;
config.st7789_gpio_reset = 23;
```

`gpio_backlight` 需要单独谨慎处理：

```cpp
config.st7789_gpio_backlight = 25;
```

但当前计划是：

```text
BLK -> 3.3V 常亮
```

这意味着代码里的 `gpio_backlight` 实际不控制背光。当前代码没有“禁用背光 GPIO”的配置，因此不要随便填 `-1`，否则现有 GPIO 操作可能失败。

更稳妥的选择：

```text
方案 A：BLK 继续接 3.3V，gpio_backlight 暂时填一个确认安全、不会影响系统的 GPIO。
方案 B：把 BLK 改接一个真实可控 GPIO，再把 gpio_backlight 配成对应编号。
```

第一版点屏建议优先确认：

```text
/dev/spidev1.0
DC = 128
RES = 23
SPI speed = 8MHz
BLK = 3.3V 常亮
```

先让屏幕稳定点亮，再处理背光 GPIO 控制和更高 SPI 速度。
