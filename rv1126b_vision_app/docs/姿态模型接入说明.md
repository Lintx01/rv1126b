# 姿态模型接入说明

本文档说明当前工程中姿态模型的接入方式、输出解析、坐标映射、姿态规则和调试方法。当前姿态模型属于三模型管线的一部分，和手势模型、水杯模型共同工作。

## 当前接入

- 模型路径：`model/yolov8n-pose-rv1126b-i8.rknn`
- 模型类型：YOLOv8-Pose
- 输入尺寸：`640x640`
- 关键点格式：COCO 17 个关键点
- 接入位置：`PoseModel::load()`、`PoseModel::infer()`、`PoseModel::parseOutput()`
- 主要代码文件：`src/PerceptionModules.cpp`
- 结果结构：`PoseResult`

`src/main.cpp` 中的默认配置：

```cpp
config.pose_model_path = "model/yolov8n-pose-rv1126b-i8.rknn";
config.pose_input_width = 640;
config.pose_input_height = 640;
config.pose_interval_ms = 150;     间隔
config.pose_keypoint_score_threshold = 0.35F;    关键点位的置信度阈值
```

姿态模型默认在三模型管线中启用：

```cpp
config.use_three_model_pipeline = true;        pipeline接口
config.use_legacy_posture_drink_model = false;         旧判别模型 ->  对应rv1126b_vision_app\src\PostureDrinkModel.cpp
```

注意：当前 AI 状态机在 Idle 时只运行手势模型。识别到 Start 手势进入 Running 后，才会调度姿态和水杯模型。调试姿态时可用：

```bash
RV_FORCE_AI_RUNNING=1 ./rv1126b_vision_app
```

## 输入预处理

摄像头当前输出 `PixelFormat::NV12`，模型输入统一由 `ImageProcessor::cropResize()` 转成 RGB888。

当前支持两种输入几何方式：

```text
resize     直接拉伸到 640x640，当前默认
letterbox  保持比例缩放，用 114 灰边补齐到 640x640   letter      
```

运行时可通过环境变量切换：

```bash
RV_PREPROCESS_MODE=opencv RV_PREPROCESS_GEOMETRY=resize ./rv1126b_vision_app
RV_PREPROCESS_MODE=opencv RV_PREPROCESS_GEOMETRY=letterbox ./rv1126b_vision_app
RV_PREPROCESS_MODE=rga RV_PREPROCESS_GEOMETRY=resize ./rv1126b_vision_app
```

`letterbox` 会记录缩放比例和补边信息，后处理结果会通过 `PreprocessTransform` 映射回原始摄像头坐标。

## 输出解析逻辑

`PoseModel::parseOutput()` 当前兼容两类 YOLOv8-Pose 输出。

### Raw DFL 输出

Rockchip / RKNN 常见 raw 输出：

```text
输出 0/1/2: [1, 65, H, W]
输出 3:     [1, 17, 3, 8400]
```

其中：

- `65 = 64 + 1`
- `64 = 4 * 16`，表示 DFL box 距离分布
- `1` 是 person 置信度
- `17` 是 COCO 关键点数量
- `3` 是每个关键点的 `x, y, score`
- `8400 = 80*80 + 40*40 + 20*20`

解析流程：

1. 前三个输出按检测 head 遍历 grid cell。
2. 对 4 个方向的 16-bin DFL 分布做 softmax。
3. 解码人体框。      
4. 使用 person score 过滤候选。
5. 按 anchor/grid cell index 读取第 4 个输出中的 17 个关键点。
6. 对人体框执行 NMS。
7. 选择最高分人体作为最终 `PoseResult`。


成功时日志 message 前缀：

```text
pose_yolov8_raw_dfl_decoder
```

### Decoded tensor 输出
`decodeYoloPoseRawOutputs`
兼容 Ultralytics 常见 decoded 输出：

```text
[1, N, 56]
[1, 56, N]
```

每个候选格式：

```text
[cx, cy, w, h, person_score, 17 * (kp_x, kp_y, kp_score)]
```

成功时日志 message 前缀：

```text
pose_yolov8_decoded_tensor
```

如果没有匹配到支持的输出布局，结果会是：

```text
pose_output_format_unmatched_raw_or_decoded_yolov8_pose
```

## PoseResult 字段

姿态模型输出统一写入 `PoseResult`：

```cpp
struct PoseResult {
    bool valid;
    bool has_person;
    uint64_t frame_id;
    int64_t timestamp_ms;
    Box person_box;
    std::array<PoseKeypoint, 17> keypoints;
    float person_score;
    std::string message;
};
```

关键含义：

- `valid=true`：模型输出成功解析并通过 NMS。
- `has_person=true`：检测到有效人体。
- `person_box`：最高分人体框。
- `keypoints`：COCO 17 点。
- `person_score`：人体框置信度。
- `message`：后处理路径和调试信息。

## 坐标映射

`PoseModel::parseOutput()` 先按模型输入坐标解析，也就是 `640x640` 坐标系。

在 `App::aiLoop()` 中，代码会调用：

```cpp
mapPoseToOriginal(last_pose_result, pose_transform);
```

映射后：

- `person_box` 变成原始摄像头坐标。
- 17 个关键点也变成原始摄像头坐标。
- `message` 会追加 `coords=original`。

后续姿态规则、喝水距离判断、网页叠框都使用原图坐标，避免 `640x480 -> 640x640` 拉伸或 letterbox 补边造成坐标误差。

## Front45 姿态判断

姿态规则在 `PostureAnalyzer` 中实现，核心函数来自 `classifyFront45Rules()`。

当前使用的关键点：

```text
nose            0
left_ear        3
right_ear       4
left_shoulder   5
right_shoulder  6
```

判断流程：

1. 鼻尖必须可见。
2. 从左耳-左肩、右耳-右肩中选择置信度更高的一侧。
3. 计算头前伸角 `head_forward_angle`。
4. 计算低头/后仰角 `head_pitch_angle`。
5. 根据固定阈值生成姿态状态。

当前阈值：

```text
head_forward_angle >= 35  -> HEAD_FORWARD
head_pitch_angle >= 15    -> HEAD_DOWN
head_pitch_angle <= -15   -> HEAD_BACKWARD
HEAD_FORWARD + HEAD_DOWN  -> CHEST_COLLAPSE_RISK
```

最终状态映射：

```text
无有效人体或关键点不足 -> PostureState::UNKNOWN
规则可判断且未异常     -> PostureState::GOOD
规则判定异常姿态       -> PostureState::BAD_ALERT
```

当前没有滑动窗口和冷却时间；`PostureAnalyzer::reset()` 保留接口，后续可加入连续帧确认、告警抑制和规则参数配置。

## 和水杯模型的关系

姿态模型输出的人体框和头部点会被 `DrinkDetector` 使用。

喝水判断当前逻辑：

1. 优先使用鼻尖作为头部位置。
2. 鼻尖不可见时，使用左右耳可见点均值。
3. 计算水杯中心到头部点的距离。
4. 用人体框对角线做归一化。
5. 距离连续命中 `drink_consecutive_hits` 次后返回 `DRINK_DETECTED`。
6. 检测到人和杯子但距离较远时返回 `NEED_REMIND`。

相关配置：

```cpp
config.pose_keypoint_score_threshold = 0.35F;
config.drink_distance_norm_threshold = 0.40F;
config.drink_consecutive_hits = 3;
```

## 接入新姿态模型时需要检查

更换姿态 RKNN 后，优先确认：

- 模型输入是否仍为 RGB 640x640。
- RKNN 内置预处理是否和训练一致。
- 输出数量是否为 4。
- 前三个输出是否是 `[1, 65, H, W]` 或等价排列。
- 第四个输出是否包含 `17 * 3 * total_anchor` 个关键点值。
- 如果是 decoded 输出，是否符合 `[1, N, 56]` 或 `[1, 56, N]`。
- `pose_yolov8_raw_dfl_decoder` 或 `pose_yolov8_decoded_tensor` 是否出现在日志中。
- `person_box` 和关键点是否已经追加 `coords=original`。

如果实际输出布局不符合以上格式，需要按真实 tensor layout 修改 `PoseModel::parseOutput()` 或新增对应解码分支。
