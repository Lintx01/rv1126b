# 水杯模型接入说明

## 当前接入

- 模型路径：`model/yolov8n_rv1126b_i8.rknn`
- 输入尺寸：`640x640`
- 目标类别：饮水容器相关 COCO 类别
- 接入位置：`CupModel::parseOutput()`

当前筛选的 YOLO/COCO 0-based 类别：

```text
39 -> bottle
40 -> wine glass
41 -> cup
45 -> bowl
```

## 解析逻辑

水杯检测现在优先走 YOLOv8 检测解析，只保留饮水容器相关类别，避免其他物体框进入喝水距离判断。

当前兼容三类输出：

1. YOLOv8 raw heads
   - 典型形状为 3 个检测头。
   - 每个检测头包含 `64` 个 DFL box 通道和 `80` 个类别通道。
   - 代码会做 DFL 解码、阈值筛选和 NMS。

2. 已解码 YOLO tensor
   - 兼容 `[N,84]` 或 `[84,N]`。
   - 每个候选框格式为 `[cx, cy, w, h, 80类分数]`。
   - 从 `39/40/41/45` 中取分数最高的类别作为饮水容器置信度。

3. 已后处理检测框
   - 兼容 `[x, y, w/h, score, class]` 或 `[x, y, w/h, score]`。
   - 如果带 class 字段，只保留 `39/40/41/45`。
   - 如果不带 class 字段，会作为最后兜底解析，日志标记为 `cup_flat_boxes_no_class_id`。

## 日志判断

上板后可以看 `CupResult::message`，常见前缀含义如下：

- `cup_yolov8_raw_decoder`：走 raw YOLOv8 检测头解析。
- `cup_yolov8_decoded_tensor`：走 `[N,84]` / `[84,N]` 解析。
- `cup_postprocessed_boxes`：走已经后处理好的框解析。
- `cup_flat_boxes_no_class_id`：走无类别信息的兜底框解析。

日志里会带：

- `class_ids=39|40|41|45`
- `candidates=...`
- `kept_after_nms=...`
- `best_score=...`

如果 `candidates=0`，优先确认模型输出 tensor layout 和置信度阈值 `cup_score_threshold`。

## 和姿态模型的关系

水杯和姿态都使用 `640x640` 输入，是为了让两边的框和关键点处在一致的模型输入尺度里，后续计算人脸、嘴部、手部和水杯之间的相对距离时更好对齐。

注意：如果原始摄像头是 `640x480`，不建议直接拉伸到 `640x640`。理想方案是 letterbox：按比例缩放到 `640x480`，上下补边到 `640x640`，后处理时再把坐标反变换回原图坐标。
