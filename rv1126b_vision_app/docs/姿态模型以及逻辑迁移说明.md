## 修改文件

### `src/PerceptionModules.cpp`

新增的姿态后处理覆盖三类情况：

1. Rockchip 风格的 YOLOv8-Pose raw 输出

   - 前 3 个输出按检测 head 处理。
   - 每个检测 head 支持 `64 + 1` 通道：
     - `64 = 4 * 16`，用于 DFL box 解码。
     - `1` 是 person 置信度。
   - 第 4 个输出按 `17 * 3` 个关键点通道处理。
   - 解码流程包括：
     - DFL softmax 解码框距离。
     - grid cell 到输入图坐标换算。
     - person 置信度筛选。
     - 读取 17 个 COCO keypoints。
     - NMS 去重。
     - 多人结果中选择最高分人体。

2. Ultralytics 常见 decoded 输出 `[1, N, 56]`

   每个候选连续存放：

   ```text
   [cx, cy, w, h, person_score, 17 * (kp_x, kp_y, kp_score)]
   ```

3. Ultralytics 常见 decoded 输出 `[1, 56, N]`

   同一通道的所有候选连续存放，也会转换成统一的 `PoseResult`。

## Front45 姿态判断

拿到人体框和 17 个关键点之后，继续使用参考工程的 Front45 规则：

- 鼻尖、耳朵、同侧肩膀用于计算头前伸角和低头角。
- `head_forward_angle >= 35` 判为头前伸。
- `head_pitch_angle >= 15` 判为低头。
- `head_pitch_angle <= -15` 判为后仰。
- 头前伸和低头同时出现时，归为 `CHEST_COLLAPSE_RISK`。

最终映射到当前工程已有状态：

- 可正常判断且无异常：`PostureState::GOOD`
- 规则判定异常姿态：`PostureState::BAD_ALERT`
- 关键点不足或置信度不足：`PostureState::UNKNOWN`

## 喝水判断

`DrinkDetector` 保持当前工程原接口不变，只增加内部状态：

- 优先使用鼻尖作为头部位置。
- 鼻尖不可见时，用左右耳可见点均值兜底。
- 计算水杯中心到头部点的距离。
- 使用人体框对角线做归一化，减少分辨率和摄像头距离影响。
- 连续命中达到 `drink_consecutive_hits` 后返回 `DRINK_DETECTED`。
- 有水杯但距离较远时返回 `NEED_REMIND`。

## 后续仍建议实机确认

虽然已经补入 raw DFL 解码和 decoded tensor 两类常见格式，但不同 RKNN 转换参数可能导致输出顺序或维度排列不同。上板后建议先打印 `RknnModel::outputInfos()` 中的输出数量和 dims，确认是否符合：

```text
raw 模式：
输出 0/1/2: [1, 65, H, W] 或等价排列
输出 3:     [1, 51, total_anchor] 或等价排列

decoded 模式：
[1, N, 56] 或 [1, 56, N]
```

如果实际 dims 不符合以上格式，需要按真实 tensor layout 微调 `PoseModel::parseOutput()` 中的读取顺序。
