# 板端预处理方案测试说明

本文档用于在 RV1126B 板端对比测试模型输入预处理方案，重点排查 cup 检测效果是否受 `resize` / `letterbox` 影响。

## 1. 当前可选方案

程序现在支持两种模型输入几何处理方式：

1. `resize`
   - 默认方案。
   - 将摄像头原图直接拉伸到模型输入尺寸。
   - 例如原图 `640x480`，cup 模型输入 `640x640` 时，会直接把画面纵向拉伸。

2. `letterbox`
   - 新增方案。
   - 保持原图宽高比例缩放，再用灰边补齐到模型输入尺寸。
   - 例如原图 `640x480`，cup 模型输入 `640x640` 时，会保持 `640x480` 内容比例，上下补边到 `640x640`。
   - 补边颜色为 `114`，这是 YOLO 系列常见预处理填充值。

两种方案都会把模型输出的框和关键点反算回原始摄像头坐标，所以网页叠框、饮水判断使用的仍然是原图坐标。

## 2. 板端运行命令

### 2.1 测试原来的 resize 方案

如果不设置环境变量，默认就是 `resize`：

```bash
./rv1126b_vision_app
```

也可以显式指定：

```bash
RV_PREPROCESS_GEOMETRY=resize ./rv1126b_vision_app
```

如果需要指定 OpenCV 预处理后端：

```bash
RV_PREPROCESS_MODE=opencv RV_PREPROCESS_GEOMETRY=resize ./rv1126b_vision_app
```

### 2.2 测试新的 letterbox 方案

推荐先用 OpenCV 后端测试效果：

```bash
RV_PREPROCESS_MODE=opencv RV_PREPROCESS_GEOMETRY=letterbox ./rv1126b_vision_app
```

如果现场希望尽量走非 OpenCV 路径，也可以只指定 letterbox：

```bash
RV_PREPROCESS_GEOMETRY=letterbox ./rv1126b_vision_app
```

注意：当前 `letterbox` 会优先走 OpenCV 实现；如果 OpenCV 不可用，会回退到软件实现。`RV_PREPROCESS_MODE=rga` 主要用于原来的直接 resize 方案。

## 3. 日志确认方式

启动后看日志里是否出现对应配置：

```text
[Config] preprocess_geometry=resize(default)
```

或：

```text
[Config] preprocess_geometry=letterbox
```

同时 `ImageProcessor` 初始化日志会带上当前几何处理方式：

```text
geometry=resize
```

或：

```text
geometry=letterbox
```

cup 模型推理后继续观察日志：

```text
[AI] cup result valid=..., message=...
```

重点看：

- `message` 是否为 `cup_yolov8_split_decoder`
- `candidates` 是否明显减少到合理范围
- `kept_after_nms` 是否明显减少
- `best_score` 是否可信
- 画面上的 cup 框是否贴合水杯

## 4. 建议测试步骤

1. 同一台摄像头、同一个人、同一个杯子、同一个距离下，先跑 `resize`。
2. 记录 10 到 20 秒日志，截图保存 cup 框效果。
3. 不移动摄像头和杯子，切到 `letterbox`。
4. 再记录 10 到 20 秒日志，截图保存 cup 框效果。
5. 对比两组结果：
   - 哪个方案更容易识别出杯子。
   - 哪个方案框更贴合。
   - 哪个方案误检更少。
   - `candidates` 和 `kept_after_nms` 是否更合理。

## 5. 判断结论参考

如果 `letterbox` 明显优于 `resize`：

- 说明模型训练或导出时更接近 YOLO 标准 letterbox 预处理。
- 后续建议默认开启 `RV_PREPROCESS_GEOMETRY=letterbox`。

如果 `resize` 明显优于 `letterbox`：

- 说明当前 RKNN 模型可能是在直接 resize 输入下导出或校准的。
- 后续可以继续保留默认 `resize`。

如果两种方案都识别不到明显水杯：

- 优先怀疑模型本身的 cup 泛化能力、类别是否为 COCO cup、RKNN 量化或输出解析是否还有问题。
- 可以再用 PC 端 `tools/check_cup_pt.py` 对同一张图片跑 `.pt` 权重，对比 `.pt` 和 `.rknn` 是否一致。
